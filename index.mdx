RKNHardering is an open-source Android application that detects whether a device is using a VPN, proxy, or other censorship circumvention tool. It implements the Roskomnadzor-style methodology: six independent detection modules run in parallel, each gathering signals from different system layers — GeoIP, network capabilities, routing tables, DNS, localhost ports, and location data — then a verdict engine combines them into a final verdict.

## How detection works

RKNHardering runs six modules in parallel every time you tap **Run Check**. Each module targets a different signal layer, so a single circumvention tool is unlikely to hide from all of them at once.

<Steps>
  <Step title="Tap Run Check">
    The app launches all six detection modules concurrently via `VpnCheckRunner`. Each module reports results as soon as it finishes — you see cards appear in real time.
  </Step>
  <Step title="Modules gather signals">
    Modules query GeoIP APIs, inspect `NetworkCapabilities`, scan network interfaces, analyze routing tables, probe localhost ports, and check telephony/cell data.
  </Step>
  <Step title="VerdictEngine calculates a verdict">
    The verdict engine applies unconditional bypass rules first, then evaluates a three-axis signal matrix (GeoIP × direct × indirect) to produce one of three verdicts.
  </Step>
  <Step title="Review your results">
    Each module card shows its status and findings. Tap **Verdict details** to see the full breakdown of what was detected and why.
  </Step>
</Steps>

## Verdict states

| Verdict | Meaning |
|---------|---------|
| `NOT_DETECTED` | No significant VPN or proxy signals found |
| `NEEDS_REVIEW` | One or more ambiguous signals — manual inspection recommended |
| `DETECTED` | Strong evidence of an active VPN, proxy, or bypass tool |
